## 요구사항

### 메뉴 그룹

- [ ] 메뉴 그룹을 생성할 수 있다
    - [ ] 이름은 반드시 존재해야 한다
    - [ ] 생성 시 고유한 식별자를 부여받는다

- [ ] 등록된 메뉴 그룹 전체를 조회할 수 있다

### 메뉴

- [ ] 메뉴를 생성할 수 있다
    - [ ] 이름은 반드시 존재해야 하며 비속어를 포함할 수 없다
    - [ ] 가격은 반드시 존재해야 하며 0 이상이어야 한다
    - [ ] 하나의 메뉴 그룹에 속해야 한다
    - [ ] 생성 시 고유한 식별자를 부여받는다
    - [ ] 메뉴 상품 구성
        - [ ] 메뉴 상품은 반드시 1개 이상 존재해야 한다
        - [ ] 메뉴 상품의 수량은 0보다 커야 한다
        - [ ] 등록된 상품만 추가할 수 있다
        - [ ] 메뉴의 가격은 메뉴 상품들의 총 가격 합계를 초과할 수 없다

- [ ] 메뉴의 가격을 수정할 수 있다
    - [ ] 메뉴는 반드시 존재하는 메뉴여야 한다
    - [ ] 가격은 0 이상이어야 한다
    - [ ] 메뉴 상품들의 총 가격 합계를 초과할 수 없다

- [ ] 메뉴의 표시 상태를 변경할 수 있다
    - [ ] 보임 상태로 변경할 수 있다
        - [ ] 메뉴는 반드시 존재하는 메뉴여야 한다
        - [ ] 메뉴의 가격은 구성된 메뉴 상품들의 총 가격보다 높을 수 없다
    - [ ] 숨김 상태로 변경할 수 있다
        - [ ] 메뉴는 반드시 존재하는 메뉴여야 한다

- [ ] 등록된 전체 메뉴를 조회할 수 있다

### 주문

- [ ] 주문을 생성할 수 있다
    - [ ] 공통 조건
        - [ ] 생성 시 고유한 식별자를 부여받는다
        - [ ] 주문 타입(배달/포장/방문)은 반드시 존재해야 한다
        - [ ] 생성 시 '대기 중' 상태가 설정된다
        - [ ] 주문 시각이 자동으로 기록된다
        - [ ] 주문 항목
            - [ ] 주문 항목은 반드시 1개 이상 존재해야 한다
            - [ ] 주문 항목의 가격은 반드시 존재해야 한다
            - [ ] 주문할 메뉴는 반드시 존재해야 한다
            - [ ] 주문할 메뉴는 판매 중 상태여야 한다
            - [ ] 주문 항목의 가격이 메뉴의 현재 가격과 일치해야 한다

    - [ ] 배달 주문
        - [ ] 배달 주소는 반드시 존재해야 한다
        - [ ] 배달 주소는 공백이 될 수 없다
        - [ ] 주문 항목의 수량은 1 이상이어야 한다
        - [ ] 상태 흐름: 대기중 -> 수락됨 -> 서빙됨 -> 배달중 -> 배달완료 -> 완료됨
        - [ ] 배달 주문 수락 시
            - [ ] 주문 총액을 계산한다
            - [ ] 배달 기사를 요청한다 (주문 번호, 총액, 배달 주소 필요)

    - [ ] 포장 주문
        - [ ] 주문 항목의 수량은 1 이상이어야 한다
        - [ ] 상태 흐름: 대기중 -> 수락됨 -> 서빙됨 -> 완료됨

    - [ ] 매장 식사 주문
        - [ ] 주문 테이블은 반드시 존재해야 한다
        - [ ] 지정된 테이블이 사용 중 상태여야 한다
        - [ ] 주문 항목의 수량 제한이 없다
        - [ ] 상태 흐름: 대기중 -> 수락됨 -> 서빙됨 -> 완료됨
        - [ ] 주문 완료 시 테이블 처리
            - [ ] 테이블의 모든 주문이 완료된 경우
                - [ ] 테이블의 손님 수를 0으로 변경한다
                - [ ] 테이블을 비어있음으로 변경한다

- [ ] 주문 상태를 변경할 수 있다
    - [ ] 주문을 수락할 수 있다.
        - [ ] 주문은 반드시 존재하는 주문이어야 한다
        - [ ] 대기 상태여야 한다

    - [ ] 서빙을 할 수 있다.
        - [ ] 주문은 반드시 존재하는 주문이어야 한다
        - [ ] 수락 상태여야 한다

    - [ ] 배달을 할 수 있다. (배달 주문만 해당)
        - [ ] 배달을 시작할 수 있다.
            - [ ] 서빙된 상태여야 한다
        - [ ] 배달을 완료할 수 있다.
            - [ ] 배달 중 상태여야 한다

    - [ ] 주문을 완료할 수 있다.
        - [ ] 주문은 반드시 존재하는 주문이어야 한다
        - [ ] 주문 타입별 완료 조건
            - [ ] 배달 주문은 배달완료 상태여야 한다
            - [ ] 포장/방문 주문은 서빙됨 상태여야 한다

- [ ] 등록된 모든 주문을 조회할 수 있다

### 주문 테이블

- [ ] 테이블을 생성할 수 있다
    - [ ] 이름은 반드시 존재해야 한다
    - [ ] 생성 시 고유한 식별자를 부여받는다
    - [ ] 초기 손님 수는 0으로 설정된다
    - [ ] 초기 상태는 비어있음으로 설정된다

- [ ] 테이블 상태를 관리할 수 있다
    - [ ] 테이블에 착석할 수 있다
        - [ ] 테이블은 반드시 존재하는 테이블이어야 한다
        - [ ] 상태를 점유됨으로 변경할 수 있다

    - [ ] 테이블을 정리할 수 있다
        - [ ] 테이블은 반드시 존재하는 테이블이어야 한다
        - [ ] 완료되지 않은 주문이 없어야 한다
        - [ ] 손님 수를 0으로 변경할 수 있다
        - [ ] 상태를 비어있음으로 변경할 수 있다

    - [ ] 테이블의 손님 수를 변경할 수 있다
        - [ ] 테이블은 반드시 존재하는 테이블이어야 한다
        - [ ] 점유된 상태여야 한다
        - [ ] 변경할 손님 수는 0 이상이어야 한다

- [ ] 등록된 모든 테이블을 조회할 수 있다

### 상품

- [ ] 상품을 생성할 수 있다
    - [ ] 이름은 반드시 존재해야 하며 비속어를 포함할 수 없다
    - [ ] 가격은 반드시 존재해야 하며 0보다 커야 한다
    - [ ] 생성 시 고유한 식별자를 부여받는다

- [ ] 상품의 가격을 변경할 수 있다
    - [ ] 상품은 반드시 존재하는 상품이어야 한다
    - [ ] 가격은 0보다 커야 한다
    - [ ] 연관된 메뉴 처리
        - [ ] 해당 상품을 사용하는 모든 메뉴를 조회할 수 있다
        - [ ] 각 메뉴의 구성 가격을 재계산할 수 있다
        - [ ] 메뉴 판매가가 구성 가격보다 낮으면 숨김 처리할 수 있다

- [ ] 등록된 모든 상품을 조회할 수 있다

## 용어 사전

### 1. 메뉴 그룹

| 한글명   | 영문명       | 설명                      |
|-------|-----------|-------------------------|
| 메뉴 그룹 | MenuGroup | 여러 메뉴들을 묶어서 관리할 수 있는 그룹 |

### 2. 메뉴

| 한글명 | 영문명  | 설명           |
|-----|------|--------------|
| 메뉴  | Menu | 주문 가능한 메뉴 항목 |

#### 2.1 메뉴 속성

| 한글명      | 영문명           | 설명                                               |
|----------|---------------|--------------------------------------------------|
| 메뉴 이름    | MenuName      | 메뉴의 이름, 필수 값이며 비속어를 포함할 수 없다.                    |
| 메뉴 가격    | MenuPrice     | 메뉴의 판매 가격. 0 이상이어야 하며 메뉴 상품들의 총 가격 합계를 초과할 수 없다. |
| 메뉴 표시 상태 | DisplayStatus | 메뉴의 표시 여부 (보임/숨김)                                |

#### 2.2 메뉴 상품

| 한글명      | 영문명                  | 설명                               |
|----------|----------------------|----------------------------------|
| 메뉴 상품    | MenuProduct          | 반드시 1개 이상 존재하는 등록된 상품만 추가할 수 있다. |
| 메뉴 상품 수량 | MenuProductQuantity	 | 1 개 이상의 메뉴 상품의 수량,               |

### 3. 주문

| 한글명 | 영문명   | 설명                                                                    |
|-----|-------|-----------------------------------------------------------------------|
| 주문  | Order | 손님이 요청한 주문, 주문 시 값 주문들의 타입(배달/포장/매장 식사)과 주문 시각이 기록되며 초기값은 [대기] 상태 이다. |

#### 3.1 주문 타입

#### 3.1.1 배달 주문

| 한글명   | 영문명           | 설명           |
|-------|---------------|--------------|
| 배달 주문 | DeliveryOrder | 주문들의 [배달] 타입 |

#### 3.1.2 포장 주문

| 한글명   | 영문명          | 설명                              |
|-------|--------------|---------------------------------|
| 포장 주문 | TakeOutOrder | 손님이 매장에서 직접 상품을 수령하는 주문 [포장] 타입 |

#### 3.1.3 매장 식사 주문

| 한글명      | 영문명        | 설명                                             |
|----------|------------|------------------------------------------------|
| 매장 식사 주문 | EatInOrder | 손님이 매장 내 지정된 테이블에서 식사하는  [매장 식사] 주문 타입         |
| 주문 테이블   | OrderTable | [매장 식사] 주문 시 [착석]하지 않은 상태의 식사 위치를 지정할 수 있는 테이블 |

#### 3.2 주문 상태

| 한글명      | 영문명        | 설명                                                                  |
|----------|------------|---------------------------------------------------------------------|
| 대기       | Waiting    | 주문이 생성된 초기 단계                                                       |
| 수락       | Accepted   | 매장에서 대기 중 상태에서 접수하여 [서빙] 단계로 이동하기 전 상태                              |
| 메뉴 준비 완료 | Served     | 주문이 수락 된 이후의 상태로서 손님에게 전달 가능한 상태                                    |
| 배달 중     | DELIVERING | [배달] 주문이 [수락]된 이후의 상태로서 손님의 주소로 배달 중인 상태                            |
| 배달 완료    | DELIVERED  | [배달] 주문이 [수락]된 이후의 [배달]의 [종료 상태]로서 배달 중 인 물품을 사용자의 주소로 배달이 [완료]된 상태 |
| 완료       | Completed  | 주문이 최종 완료 상태로서 주문의 모든 프로세스를 종료한다.                                   |

### 4. 주문 테이블

| 한글명    | 영문명        | 설명                                             |
|--------|------------|------------------------------------------------|
| 주문 테이블 | OrderTable | [매장 식사 주문] 시 [착석]하지 않은 상태의 식사 위치를 지정할 수 있는 테이블 |

### 4.1 주문 테이블 속성

| 한글명 | 영문명        | 설명                             |
|-----|------------|--------------------------------|
| 착석  | Occupied   | [주문 테이블]에 1 명 이상의 손님이 [착석]한 상태 |
| 미착석 | UnOccupied | [주문 테이블]에 어떠한 손님도 [착석]하지 않은 상태 |

### 5. 상품

| 한글명 | 영문명     | 설명                                      |
|-----|---------|-----------------------------------------|
| 상품  | Product | 비속어를 포함할 수 없는 이름을 갖는 판매할 수 있는 상품의 최소 단위 |

### 5.1 상품 속성

| 한글명   | 영문명          | 설명                                |
|-------|--------------|-----------------------------------|
| 상품 가격 | ProductPrice | 상품의 구매 가격으로서 0보다 큰 가격만을 입력할 수 있다. |

### 6 손님

| 한글명 | 영문명      | 설명                                                     |
|-----|----------|--------------------------------------------------------|
| 손님  | Customer | 매장에서 배달/포장/매장 식사 형태로 주문할 수 있는 주체로, 테이블 착석 및 주문이 가능한 대상 |
