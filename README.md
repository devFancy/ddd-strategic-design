# 키친포스

## 퀵 스타트

```sh
cd docker
docker compose -p kitchenpos up -d
```

## 요구사항

### 메뉴
- 메뉴를 등록한다.
  - [ ] 메뉴 가격은 0원 이상이어야 한다.
  - [ ] 메뉴는 메뉴 그룹에 속해야 한다.
  - [ ] 메뉴는 1개 이상의 등록된 상품으로 구성된다.
  - [ ] 메뉴 생성 시 지정된 모든 상품이 실제로 등록되어 있어야 한다.
  - [ ] 메뉴 상품 수량은 0개 이상이어야 한다.
  - [ ] 메뉴 가격은 구성 상품의 가격 총 합보다 큰 경우 등록할 수 없다.
  - [ ] 메뉴 이름에는 비속어가 포함될 수 없다
- 메뉴 가격을 변경할 수 있다
  - [ ] 메뉴 가격은 0원 이상 이어야 한다
  - [ ] 메뉴 가격의 합이 구성 상품의 가격 총 합보다 큰 경우 수정할 수 없다.
- 메뉴를 노출 할 수 있다.
  - [ ] 메뉴의 실제 가격이 구성 상품들의 총 가격보다 작거나 같으면 노출 할 수 있다
- 메뉴를 숨길 수 있다.
  - [ ]  메뉴가 존재 하지 않는 경우 숨김 상태로 변경할 수 없다
- [ ] 메뉴를 조회할 수 있다

### 메뉴 그룹
- 메뉴 그룹을 등록한다.
  - [ ] 메뉴 그룹 이름은 빈 값이면 안되고, 필수이다.
- [ ] 메뉴 그룹을 조회할 수 있다.


### 상품
- 상품을 등록 한다.
  - [ ] 상품 가격은 0원 이상 이어야 한다.
  - [ ] 상품 이름에는 비속어가 포함될 수 없다.
- 상품을 가격을 수정 한다.
  - [ ] 상품 가격은 있어야 하고, 0원 이상 이어야 한다.
  - [ ] 메뉴 가격이 상품의 합산 가격 보다 클때 숨길 수 있다.
- [ ] 상품을 조회 한다.


### 주문
- 주문을 생성한다.
  - [ ] 주문은 반드시 주문 유형을 가지고 있어야 한다 (DELIVERY, TAKEOUT, EAT_IN).
  - [ ] 주문은 한 개 이상의 주문 상품을 가지고 있어야 한다.
  - [ ] 주문 항목의 메뉴는 실제로 존재해야 한다.
  - [ ] 매장 식사가 아닌 경우에는 수량이 음수일 수 없다.
    - 매장식사의 경우는 음수수량을 허용한다.
  - [ ] 메뉴가 존재하고, 메뉴가 노출 되어있는 경우에만 주문할 수 있다.
  - [ ] 요청된 주문 가격이 메뉴 가격과 일치해야 한다.
  - 배달 주문
    - [ ] 배달 주문의 경우 주소가 필수이다.
  - 매장 주문
    - [ ] 매장 주문의 경우 테이블 번호가 필수이다.
    - [ ] 테이블을 고객이 사용중 이어야 한다.
  -
- 주문 상태 관리
  - 주문을 접수한다.
    - [ ] 대기(WAITING) 상태의 주문만 접수할 수 있다.
    - [ ] 배달 주문(DELIVERY)의 경우 배달 서비스에 배달 요청을 한다.
    - [ ] 주문 접수 시 주문 상태는 접수(ACCEPTED)로 변경된다.
  - 주문을 서빙한다.
    - [ ] 주문 상태가 접수(ACCEPTED)인 주문만 서빙할 수 있다.
    - [ ] 주문 서빙 시 주문 상태는 서빙(SERVED)로 변경된다.
  - 주문을 배달한다.
  - - [ ] 주문 상태가 서빙(SERVED)인 주문만 배달할 수 있다.
  - [ ] 주문 타입이, 배달(DELIVERY)인 주문만 배달할 수 있다.
  - [ ] 주문 배달 시 주문 상태는 배달(DELIVERING)로 변경된다.
  - 배달을 완료한다.
    - [ ] 주문 상태가 배달 중(DELIVERING)인 주문만 배달 완료(DELIVERED)할 수 있다.
  - 주문을 완료한다.
    - [ ] 주문 타입이 배달(DELIVERY)인 경우, 주문 상태가 배달 완료(DELIVERED)된 주문만 완료할 수 있다.
    - [ ] 주문 타입이 매장식사(EAT_IN)와 테이크아웃(TAKEOUT)인 경우 주문 상태가 서빙 완료(SERVED)인 경우만 완료할 수 있다.
    - [ ] 매장식사 주문이 완료되면 해당 테이블의 주문이 모두 완료된 경우 테이블을 초기화한다
- [ ] 주문을 조회한다.

### 주문 테이블
- 주문 테이블을 등록한다.
  - [ ] 테이블 이름은 빈 값이면 안되고, 필 수이다.
  - [ ] 초기 손님 수는 0명으로 설정 하고, 주문 테이블 상태는 비어있음으로 설정한다.
- 주문 테이블 관리
  - 주문 테이블 상태를 비어있지 않음으로 변경한다.
    - [ ] 주문 테이블이 존재해야 한다.
  - 주문 테이블 상태를 비어있음으로 변경한다.
    - [ ] 주문 테이블이 존재해야 한다.
    - [ ] 주문 상태가 완료(COMPLETED)인 주문 테이블만 비어있음으로 변경할 수 있다.
  - 주문 테이블의 손님수 변경
    - [ ] 주문 테이블이 존재 해야 한다.
    - [ ] 주문 테이블의 손님 수는 0명 이상 이어야 한다.
    - [ ] 사용 중인 테이블의 손님 수만 변경할 수 있다/
- [ ] 주문 테이블을 조회 한다.

## 용어 사전

### 공통 용어

| 한글명         | 영문명       | 설명                                                  | 
|-------------|-----------|-----------------------------------------------------|
| 포스          | POS       | Point of Sale, 매장에서 주문을 받고 결제하는 시스템                 |
| 비속어         | Profanity | 상황에 따라 불쾌감을 주거나 무례한 말                               |
| 라이더         | Rider     | 배달 주문을 배달하는 사람                                      |
| 가게 이용 고객   | Guests    | 가게에서 주문을 하는 주체자                                     |
| 외부 대행 업체   | Clients   | 키친 포스와 관련된 외부 대행 업체 혹은 외부시스템 (예: 비속어 검증, 라이더 대행상 등) |

### 메뉴

| 한글명      | 영문명           | 설명                                   |
|----------|---------------|--------------------------------------|
| 메뉴       | Menu          | 가게에서 판매하는 완성된 음식 항목. 하나 이상의 상품으로 구성됨 |
| 메뉴 이름    | Name          | 메뉴의 이름                               |
| 메뉴 가격    | Price         | 메뉴의 가격                               |
| 메뉴 노출 여부 | Menu Display | 메뉴를 노출                            | 
| 메뉴 노출 여부 | Menu Hide     | 메뉴를 숨김                               | 
| 메뉴 상품 목록 | Menu Products | 메뉴에 속한 상품 목록                         |

### 메뉴 그룹

| 한글명      | 영문명        | 설명                               |
|----------|------------|----------------------------------|
| 메뉴 그룹    | Menu Group | 메뉴들을 분류하는 카테고리 (예: 메인 요리, 디저트 등) |
| 메뉴 그룹 이름 | Name       | 메뉴 그룹의 이름                        |

### 메뉴 상품

| 한글명      | 영문명          | 설명                                         |
|----------|--------------|--------------------------------------------|
| 메뉴 상품    | Menu Product | 메뉴에 속한 상품 (예: 코카콜라, 양념반 후라이드반 -> 콜라 치킨 세트) |
| 메뉴 상품 수량 | Quantity     | 메뉴에 속한 상품의 수량                              |

### 상품

| 한글명   | 영문명     | 설명                                                       |
|-------|---------|----------------------------------------------------------|
| 상품    | Product | 메뉴를 구성하는 최소 단위, 업주가 팔고자하는 음식을 의미 (예: 코카콜라, 치킨, 양념치킨, 소주) |
| 상품 이름 | Name    | 상품의 이름, 상품을 구별하는 것                                       |
| 상품 가격 | Price   | 상품의 가격, 상품 1개의 가격                                        |

### 주문 공통

| 한글명   | 영문명             | 설명              |
|-------|-----------------|-----------------|
| 주문 항목 | Order Line Item | 주문에 속한 메뉴와 수량   |
| 주문 유형 | Order Type      | 주문의 유형          |

### 매장 테이블

| 한글명            | 영문명              | 설명               |
|----------------|------------------|------------------|
| 매장 테이블         | Order Table      | 고객이 식사를 하는 테이블   |
| 매장 테이블 이름      | Name             | 매장 테이블의 이름       |
| 매장 테이블 방문 손님 수 | Number Of Guests | 매장 테이블에 앉아있는 손님 수 |
| 매장 테이블 사용 중    | Occupied         | 매장 테이블이 비어있는지 여부 |
| 매장 테이블 비어있음    | Unoccupied       | 매장 테이블이 비어있는지 여부 |
| 매장 테이블 청소      | Clear            | 매장 테이블을 비운다      |
| 매장 테이블에 손님 착석  | Sit              | 매장 테이블에 손님이 앉는다. |

| 한글명         | 영문명                  | 설명             |

| 배달 주문       | Delivery Order       | 고객이 배달을 요청한 주문 |

| 한글명      | 영문명        | 설명              |
|----------|------------|-----------------|
| 접수 대기    | Waiting    | 배달 주문이 접수 대기 중  |
| 접수       | Accepted   | 배달 주문이 접수됨      |
| 배달 준비 완료 | Served     | 배달 주문이 요리 준비 완료 |
| 배달 중     | Delivering | 배달 주문이 배달 중     |
| 배달 완료    | Delivered  | 배달 주문이 배달 완료    |
| 주문 완료    | Completed  | 배달 주문이 완료됨      |
